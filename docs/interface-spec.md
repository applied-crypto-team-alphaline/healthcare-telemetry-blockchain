# API and Protocol Specification

Version: `v1.0-implemented-prototype`  
Status: Updated to match current implementation

## 1. Registry Record

Current device state is derived from the latest event for a device identifier.

Required fields:

- `device_id`: string
- `public_key`: hex string
- `timestamp`: number
- `status`: `active` | `revoked`

Meaning:

- `device_id`: unique simulated device identity
- `public_key`: Ed25519 public key used for authentication verification
- `timestamp`: latest update time for the record
- `status`: trust status for runtime validation

## 2. Ledger Event Record

The registry stores append-only trust events.

Required fields:

- `event_type`: `register` | `revoke`
- `device_id`: string
- `status`: `active` | `revoked`
- `public_key`: hex string
- `timestamp`: number
- `prev_hash`: hex string
- `event_hash`: hex string

Meaning:

- `event_type`: trust operation recorded in the ledger
- `status`: resulting trust status after the event
- `prev_hash`: hash of the previous event or genesis hash
- `event_hash`: hash of the current canonicalized event payload

## 3. Authentication Handshake

### 3.1 Challenge Input

Generated by receiver during authentication.

Required fields:

- `device_id`: string
- `challenge`: hex string
- `sender_ephemeral_pub_hex`: hex string
- `receiver_ephemeral_pub_hex`: hex string

Meaning:

- `device_id`: claimed sender identity
- `challenge`: fresh receiver-generated nonce
- `sender_ephemeral_pub_hex`: sender X25519 ephemeral public key
- `receiver_ephemeral_pub_hex`: receiver X25519 ephemeral public key

### 3.2 Authentication Proof

Generated by sender.

Required fields:

- `device_id`: string
- `challenge`: hex string
- `signature`: hex string
- `sender_ephemeral_pub_hex`: hex string
- `receiver_ephemeral_pub_hex`: hex string

Meaning:

- `signature`: Ed25519 signature over the bound authentication message

Socket demo note:

- in the real P2P flow, the sender first transmits a `hello` message
- the receiver responds with a `challenge` message
- the sender replies with a `proof` message

Verification rules:

- sender must exist in registry
- sender must have `active` status
- signature must verify against the registry public key

## 4. Session Establishment Output

Required fields:

- `sender_ephemeral_pub_hex`: hex string
- `receiver_ephemeral_pub_hex`: hex string
- `verified`: boolean

Meaning:

- successful authentication is completed before the session is considered valid
- session keys are derived from X25519 shared secret plus HKDF-SHA256

## 5. Telemetry Message Format

### 5.1 Plaintext Telemetry

Required fields:

- `device_id`: string
- `patient_id`: string
- `sequence_number`: integer
- `heart_rate`: integer
- `oxygen_level`: integer
- `temperature`: number
- `blood_pressure`: string
- `timestamp`: string
- `ip_address`: string
- `access_type`: string
- `action`: string
- `target`: integer

### 5.2 Encrypted Telemetry Output

Required fields:

- `nonce`: hex string
- `ciphertext`: hex string
- `algorithm`: string

Current algorithm:

- `AES-GCM`

## 6. Acceptance Rules

Telemetry is accepted only if:

- `telemetry.device_id == sender`
- sender is registered
- receiver is registered
- sender status is `active`
- receiver status is `active`
- signature proof is valid
- sequence number is not replayed
- encrypted payload is generated after successful authentication

## 7. Error Handling

Standard error outcomes:

- `ERR_DEVICE_NOT_REGISTERED`
  - when sender or receiver does not exist in registry

- `ERR_DEVICE_REVOKED`
  - when device status is not `active`

- `ERR_AUTH_PROOF_FAILED`
  - when signature verification fails

- `ERR_SPOOFING_DETECTED`
  - when claimed sender does not match telemetry sender field

- `ERR_REPLAY_DETECTED`
  - when the same `(sender, sequence_number)` pair is seen again

Prototype note:

- current implementation raises Python exceptions with matching error meaning
- the real socket demo returns structured rejection responses such as:
  - `ERR_DEVICE_NOT_REGISTERED`
  - `ERR_DEVICE_REVOKED`
  - `ERR_AUTH_PROOF_FAILED`
  - `ERR_REPLAY_DETECTED`
